## Phase 2: Fix Route-Template Issues

### 2.1 Fix Profile Route (`src/routes/profile.js`)

**Current code:**
```javascript
router.get('/', isAuthenticated(auth), async (req, res) => {
  // ... renders profile/edit.html
});
```

**Required changes:**
```javascript
// GET /profile - View profile
router.get('/', isAuthenticated(auth), async (req, res) => {
  let profile = await UserProfile.findOne({ userId: req.userId })
    .populate('userId', 'name email image');
  if (!profile) {
    profile = new UserProfile({ userId: req.userId, role: 'candidate' });
    await profile.save();
  }
  res.render('profile/view', { profile });
});

// GET /profile/edit - Edit profile
router.get('/edit', isAuthenticated(auth), async (req, res) => {
  let profile = await UserProfile.findOne({ userId: req.userId })
    .populate('userId', 'name email image');
  if (!profile) {
    profile = new UserProfile({ userId: req.userId, role: 'candidate' });
    await profile.save();
  }
  res.render('profile/edit', { profile });
});
```

## Phase 4: Workflow Improvements

### 4.1 Admin Workflow

```
Sign In (Admin)
    ↓
Admin Dashboard (/dashboard/admin)
    ├── Quick Stats: Jobs to approve, Companies to verify
    ├── Link: /admin/jobs/pending → Pending Jobs page
    ├── Link: /admin/companies/pending → Pending Companies
    └── Link: /admin/users → User Management
```

**Dashboard enhancement** (`dashboard/admin.html`):
- Add "Quick Actions" section with cards linking to:
  - Pending Jobs (with count badge)
  - Pending Companies (with count badge)
  - User Management

### 4.2 Employer Workflow

```
Sign In (Employer)
    ↓
Check: Has Company?
    ├── NO → Redirect to /company/create
    └── YES → Employer Dashboard (/dashboard/employer)
        ├── My Company profile
        ├── My Jobs list
        ├── View Applications
        └── Find Candidates (/matches)
```

**Dashboard enhancement** (`dashboard/employer.html`):
- Add "Quick Actions" cards:
  - Post New Job → /jobs/new (create form)
  - View Applications → /applications/employer
  - Update Company → /company/{id}/edit
  - Find Candidates → /matches

### 4.3 Candidate Workflow

```
Sign In (Candidate)
    ↓
Check: Profile Complete?
    ├── NO → Redirect to /profile?complete=true
    └── YES → Candidate Dashboard (/dashboard/candidate)
        ├── Recommended Jobs
        ├── Application Status
        └── Job Matches
```

**Dashboard enhancement** (`dashboard/candidate.html`):

## Phase 5: Security Best Practices Audit

### 5.1 Middleware Verification

| Route | Handler | Middleware |
|-------|---------|------------|
| `GET /profile` | View | `isAuthenticated` |
| `GET /profile/edit` | Edit | `isAuthenticated` |
| `POST /profile` | Update | `isAuthenticated + CSRF` |
| `POST /jobs` | Create | `isAuthenticated + isEmployer + CSRF` |
| `PUT /jobs/:id` | Update | `isAuthenticated + isEmployer + owner + CSRF` |
| `DELETE /jobs/:id` | Delete | `isAuthenticated + isEmployer + owner + CSRF` |
| `POST /applications` | Apply | `isAuthenticated + CSRF` |
| `PUT /applications/:id/status` | Status | `isAuthenticated + isEmployer + owner` |
| `GET /admin/*` | Admin | `isAuthenticated + isRole('admin')` |
| `POST /admin/users/:id/role` | Role | `isAuthenticated + isRole('admin') + CSRF` |

### 5.2 CSRF Protection Implementation

Ensure all POST forms include:
```html
<input type="hidden" name="_csrf" value="{{ csrfToken }}">
```

### 5.3 Input Validation

All form submissions must use express-validator with proper sanitization.
