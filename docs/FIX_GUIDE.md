# JobHub Integration & Fix Guide

## 1. Project Structure Consolidation
**Issue:** Two entry points exist (`/index.js` and `/src/index.js`).
**Action:** 
- Use `/src/index.js` as the sole entry point.
- Delete the redundant `index.js` in the root directory.
- Ensure all `package.json` scripts point to `./src/index.js`.

## 2. Security & Input Sanitization
**Issue:** `validator.escape()` was corrupting user passwords.
**Fix:** Modified `validateInput` in `src/middleware/auth.js` to skip escaping for the following fields:
- `password`
- `confirmPassword`
- `newPassword`
- `currentPassword`

## 3. Better-Auth Middleware Sequence
**Requirement:** For Better-Auth to handle POST requests correctly in a Node environment:
1. **CORS** must be initialized first.
2. **toNodeHandler(auth)** must be mounted *before* `express.json()` or `express.urlencoded()`.
3. Body parsers should be mounted *after* the auth splat route to avoid interference with Better-Auth's internal stream parsing.

## 4. CSS & MIME Type Resolution
**Issue:** Browser refusing to apply styles due to `text/html` MIME type.
**Cause:** This happens when the server returns a 404 (HTML error page) for a CSS request.
**Fix:**
- Ensure `express.static` is mounted early in `src/index.js`.
- Use absolute paths in `layout.html` (e.g., `<link rel="stylesheet" href="/css/output.css">`).
- Standardize output to `/css/output.css` to match `package.json` build scripts.
